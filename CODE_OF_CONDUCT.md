# Code of Conduct

Ledger contributors are expected to be practical, respectful, and privacy-aware.

## Expected Behavior

- Be kind and direct.
- Assume good intent, but verify facts with code and tests.
- Keep feedback focused on the project, not the person.
- Treat financial data and local machine details as sensitive by default.
- Help make the app more useful for normal weekly money decisions.

## Unacceptable Behavior

- Sharing private financial data, secrets, account numbers, or personal details.
- Harassment, insults, or exclusionary behavior.
- Pressuring contributors to expose personal data for debugging.
- Adding AI features that can mutate financial data without explicit proposal
  review and human approval.

## Enforcement

Issues or pull requests that expose private data may be edited, closed, or
removed. Repeated disrespectful behavior may result in being blocked from the
project.
